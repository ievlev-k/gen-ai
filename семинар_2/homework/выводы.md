# Выводы по домашнему заданию — Семинар 2

В первом проге произошел выраженный mode collapse: «программист» занял 66% из 50 заявок, 4 специальности не попали в выборку вовсе. Для борьбы со специализациями добавил `seed_speciality`: при каждом запросе случайно выбираемая специальность жёстко задаётся в user-промпте. Распределение выровнялось — максимум упал с 66% до 14%, все 10 специальностей представлены.

`@field_validator` не поймал ни одной ошибки за оба прогона (50/50 валидных заявок в каждом). Модель с первого раза выдавала значения в рамках ограничений. Причина — `llm_client.py` подставляет JSON Schema прямо в системный промпт, и модель уже на уровне генерации соблюдает диапазоны и перечисления. То есть валидаторы работают как страховка, а не как активный фильтр.
